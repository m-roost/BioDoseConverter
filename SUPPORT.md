# Support this project

Thanks for using this software. If it helps your work, here are ways you can support ongoing development and maintenance.

## 1) Cite the project
If you publish work that uses this code, please cite:

- This repository (see `CITATION.cff`)
- Any related papers listed in `CITATION.cff` under `references`

## 2) Tell us how you’re using it
Success stories help us justify maintenance and prioritize features.

- Open an issue titled **“Usage”** with a short description (no sensitive data), or
- Email: cmayo@med.umich.edu 

## 3) Contribute improvements
We welcome:
- Bug reports and reproducible examples
- Documentation improvements
- Pull requests (see `CONTRIBUTING.md` if present)

If you plan a larger change, please open an issue first to coordinate.

## 4) Funding / sponsorship (optional, appreciated)
If this software materially supports a funded project (e.g., a grant, contract, or paid engagement), we **request** that you consider allocating support for continued development and maintenance.

Common ways teams support this project:
- Include developer effort in a grant budget (maintenance, features, validation, documentation)
- Sponsored research collaboration or subaward
- Paid support/feature development agreement
- Institutional sponsorship

To discuss support options:
- Primary contact: Charles Mayo (cmayo@med.umich.edu)
- Co-contact: Andrew Bennet(andbenne@umich.edu)

## 5) Alternative/commercial licensing
Alternative licensing may be available for organizations that need terms different from GPL-3.0. Terms and pricing are negotiated case-by-case.

## Use of licensing revenue
Any net proceeds from commercial licensing will be directed to support ongoing development, maintenance, documentation, and user support for this project through one or more of the following channels, as applicable:
- The Regents of the University of Michigan (University of Michigan) (contact Charles Mayo) 
- NRG Oncology (contact Ying Xiao)
- AAPM (American Association of Physicists in Medicine) (contact Charles Mayo) 

Note: Listing these organizations is for transparency about intended use of funds and does not imply endorsement of this software by them.

## 6) Acknowledgment
If you’d like, we’re happy to acknowledge supporters in `README.md` or `CONTRIBUTORS.md` (subject to your approval and any funding disclosure requirements).
